Example 3: Breakpoints

what are breakpoints? as we have discussed the mobile first approach at the start of this article, so in order to achieve responsiveness for all screen sizes, starting from mobile to large screen sizes,we have some breakpoints.

xs (< 567) phones & small screens
sm (>= 567) large phones & small tablets
md (>= 768) tablets
l (>= 992) small laptops & larger tablets in landscape orientation
xl (>= 1200) large computer screens
xll (>= 1400) extra large computer screens

we can customize the number of columns in a row depending on the screen we want, when the specific breakpoint is reached the columns are placed side by side, and when the viewport width is less than the specified breakpoint, the columns are stacked on one another. now let us learn some of the magics of breakpoints

magic number 1: Breakpoints with container

before starting the magician technique of breakpoints on container, we'll leran about different types of containers. following are 2 types of containers

Tip Number 1

The Common Method of mentioning the breakpoints with properties is using the following simple convention:

attribute+breakpoint+value

Examples:

1) d-md-none (at medium sized screen display is none)
2) col-md-6 (at medium sized screen column span will be 6)
3) p-md-3 (at medium sized screen padding will be 3 [mentioned on bootstrap scale] )
3) m-md-3 (at medium sized screen margin will be 3 [mentioned on bootstrap scale] )
